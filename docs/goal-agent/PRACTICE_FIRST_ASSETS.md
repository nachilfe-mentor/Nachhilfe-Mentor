# Practice-First SEO Assets

Practice-first assets are Goal Agent-owned non-blog SEO assets. They are separate from normal blog posts and are not written by the Blog Agent.

## Asset Types

Supported `asset_type` values:

- `practice_page`
- `quiz`
- `mini_test`
- `worksheet`
- `simulator`
- `visualizer`
- `calculator`

Practice-first formats include:

- exercise pages with solutions;
- mini tests;
- quizzes;
- exam practice pages;
- grammar drills;
- formula practice pages;
- step-by-step calculation exercises;
- mistake-detection exercises;
- flashcard sets;
- worksheet generators;
- oral exam practice pages;
- topic diagnosis pages;
- typical mistakes practice pages.

## Required Fields

Each practice task/page should define:

- `asset_type`
- `subject`
- `grade_level`
- `exam_type`
- `difficulty`
- `solution_mode`
- `interaction_type`
- `expected_learning_outcome`
- `topic_cluster`
- `primary_keyword`
- `search_intent`
- `tracking_requirements`
- `schema_requirements`

## Prioritization Rules

Prioritize practice pages when:

- query intent includes `Uebungen`, `mit Loesungen`, `Aufgaben`, `Test`, `Klausur`, `Abi`, or `Arbeitsblatt`;
- SERPs are mostly static text or PDFs;
- the topic has high app conversion potential;
- the page can naturally lead to the app;
- the topic is already supported by existing topical authority.

## Quality Rules

No thin autogenerated exercise pages.

Every practice page must include:

- real educational value;
- clear tasks;
- clear solutions or explanations;
- difficulty progression;
- metadata;
- internal links;
- privacy-safe tracking;
- a natural app CTA only when useful.
- correct German visible text with real umlauts such as `ü`, `ö`, and `ä`; replacements like `ue`, `oe`, and `ae` are allowed only where they are genuinely correct German spelling or technically required in slugs, filenames, IDs, or URLs.

The Goal Agent quality gate blocks generated practice assets without solutions, difficulty metadata, interactive controls, or sufficient content.

## Adaptive Publishing

Practice assets are not limited by an arbitrary daily quota. The Goal Agent can publish/index many assets only when each one clears strict quality, uniqueness, SEO-risk, metadata, schema, tracking, and learning-outcome gates.

When evidence is promising but not strong enough, the asset should be generated or queued as draft/noindex. When duplicate intent, cannibalization, thinness, low engagement, weak indexing, or high risk appears, the asset should be held.
