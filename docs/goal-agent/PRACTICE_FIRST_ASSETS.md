# Practice-First SEO Assets

Practice-first assets are Goal Agent-owned non-blog SEO assets. They are separate from normal blog posts and are not written by the Blog Agent.

They must be real Lernmaterialien. A page is not good enough because it has a select box, a slider, or a short checklist. It must help a learner practice a concrete skill and improve after mistakes.

Public paths:

- Published, indexable assets: `/lernmaterialien/<slug>.html`
- Local drafts: `/lernmaterialien/entwuerfe/<slug>.html`
- Local simulations/prototypes: `/lernmaterialien/lernsimulationen/<slug>.html`

Never use internal implementation names in URLs or visible copy.

## Asset Types

Supported `asset_type` values:

- `practice_page`
- `quiz`
- `mini_test`
- `worksheet`
- `simulator`
- `visualizer`
- `calculator`

Practice-first formats include:

- math step trainers, for example Ableitungs-Trainer, Gleichungen lösen, Bruchrechnung, Kurvendiskussion steps;
- vocabulary or grammar trainers with active recall, correction, and repeat scheduling;
- text-analysis drills that ask learners to identify evidence, argument type, rhetorical device, thesis, or structure;
- guided German writing practice pages for open skills such as Bildbeschreibung, Argumentation, Interpretation or Erörterung when they include a concrete prompt/image, writing field, scaffold, self-check, model solution, rubric, typical mistakes and revision workflow;
- exam simulations with timed tasks, rubric feedback, and targeted retry;
- mistake-correction trainers with realistic wrong answers and explanations;
- worksheet generators only when the worksheet output is actually useful;
- flashcard sets only when they include active input, not just flip cards;
- visualizers or calculators only when they teach a concept, not just compute an answer.

Avoid tiny commodity widgets unless the page can outperform common SERP results by giving better learning feedback, stronger examples, or a more useful workflow.

## Required Fields

Each practice task/page should define:

- `asset_type`
- `subject`
- `grade_level`
- `exam_type`
- `difficulty`
- `solution_mode`
- `interaction_type`
- `expected_learning_outcome`
- `topic_cluster`
- `primary_keyword`
- `search_intent`
- `tracking_requirements`
- `schema_requirements`

## Prioritization Rules

Prioritize practice pages when:

- query intent includes `Übungen`, `mit Lösungen`, `Aufgaben`, `Test`, `Klausur`, `Abi`, or `Arbeitsblatt`;
- query intent implies an active skill, for example `Ableitung üben`, `Vokabeln lernen`, `Grammatik Übungen`, `Textanalyse üben`, `Gleichungen lösen`;
- SERPs are mostly static text or PDFs;
- the topic has high app conversion potential;
- the page can naturally lead to the app;
- the topic is already supported by existing topical authority.

## Quality Rules

No thin autogenerated exercise pages.

Every practice page must include:

- real educational value;
- active learner input;
- answer checking or rubric checking;
- immediate feedback;
- hints or step-by-step explanations;
- mistake handling;
- repeated practice, progress, or retry logic;
- clear tasks and clear solutions or explanations;
- difficulty progression;
- a concrete target learner and learning outcome;
- metadata;
- internal links;
- schema;
- privacy-safe tracking;
- a natural app CTA only when useful.
- correct German visible text with real umlauts such as `ü`, `ö`, and `ä`; replacements like `ue`, `oe`, and `ae` are allowed only where they are genuinely correct German spelling or technically required in slugs, filenames, IDs, or URLs.

Guided writing pages must not pretend to auto-grade open writing. They must help learners compare and revise their own answer through a model solution, rubric and self-check. If they use generated images, each image needs metadata and cost tracking: model, quality, size, prompt, count, estimated cost, alt text and QA status.

The Goal Agent quality gate blocks generated practice assets without active input, solutions, difficulty metadata, useful feedback, mistake handling, repeated practice, interactive controls, or sufficient content.

## Design Standard

Learning assets must look and behave like part of Nachhilfe Mentor:

- modern responsive layout that works on mobile and desktop;
- stable controls and no text overflow;
- correct tab favicon using root favicon assets;
- no broken buttons, dead controls, or fake interactivity;
- no generic landing-page hero for a learning tool;
- no large empty hero area; the exercise, prompt or image should appear quickly in the first viewport;
- no thin SEO text wrapped around a weak widget;
- clear visual hierarchy, restrained brand-consistent styling, and accessible form labels.

## Multi-Cycle Rule

If a useful learning asset cannot be completed in one coding cycle, split it:

1. Keyword and search-intent research, including SERP usefulness gap.
2. Learning design spec: target learner, task model, feedback model, scoring, retry/repetition logic.
3. Local prototype under `/lernmaterialien/lernsimulationen/` or noindex draft under `/lernmaterialien/entwuerfe/`.
4. QA on mobile and desktop, including interaction checks and visible German copy.
5. Promotion to `/lernmaterialien/` only after quality and SEO gates pass.

Do not publish a weak page just to finish a cycle.

## Adaptive Publishing

Practice assets are not limited by an arbitrary daily quota. The Goal Agent can publish/index many assets only when each one clears strict quality, uniqueness, SEO-risk, metadata, schema, tracking, design, and learning-outcome gates.

When evidence is promising but not strong enough, the asset should be generated or queued as draft/noindex. When duplicate intent, cannibalization, thinness, low engagement, weak indexing, weak usefulness, broken design, or high risk appears, the asset should be held.

Indexable learning assets should appear in Google. Add them to sitemap only after promotion to `/lernmaterialien/`, removal of noindex, canonical correctness, schema correctness, and final QA.
